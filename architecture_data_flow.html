Talk-to-Data System Architecture

Enterprise AI-Powered BI Chatbot Architecture & Data Flow Reference Map

Authentication & Onboarding Flow

  • Client Submission: The login page splits register/login forms, submitting values to backend endpoints POST /api/v1/auth/register and POST /api/v1/auth/login.
  • Cryptographic Hashing: Passwords are encrypted using bcrypt hashing protocols. Safe fallback configurations accommodate SSO integrations (nullable passwords).
  • Stateless Token Emission: A verified login issues a signed JWT token stored inside a secure HttpOnly, SameSite=Lax cookie. No token is readable via Javascript, neutralizing XSS hijacking.
  • RBAC Access Restrictor: FastAPI routes extract the cookie session, validating permissions and mapping users to respective database roles and geographic restriction criteria.

LangGraph Execution Workflow

  • Query Submission: The user submits queries to POST /api/v1/chat/message. The backend instantiates a state graph pipeline.
  • Node 1 (Guardrails): Evaluates scope by reading the current prompt and conversational history, blocking irrelevant queries.
  • Node 2 (RBAC Policy Injection): Retrieves the user's registered rules (e.g. geographic_region = 'US') and appends them as system rules.
  • Node 3 (Text-to-SQL): Compiles target schemas and history, generating standard PostgreSQL statements based on the yaml selection.
  • Node 4 (SQL Validation): Screens query blocks, strictly enforcing SELECT statements and blocking database modification syntax.
  • Node 5 & Node 6 (DB Execution & Synthesis): Runs database query execution on connection pools and formats the output into clean summaries.

Telemetry Collection & Aggregations

  • Prompt & Completion Tracking: Captures precise token usage from Google Gemini generative outputs for every query transaction.
  • Latency Calculations: Measures round-trip duration from initial client endpoint connection to session closure in milliseconds, representing it in decimal seconds (`s`) in the client browser.
  • TPS Throughput Calculations: Calculates throughput using the formula: total_tokens / latency_seconds.
  • Telemetry Storage: Saves all computed metrics, active provider configurations, and SQL blocks into the database table execution_log.

Performance Analytics Dashboard

  • Admin Analytics Route: Features a secure, admin-only endpoint GET /api/v1/admin/analytics that performs aggregate reads on telemetry logs.
  • DB Scalar Aggregations: Computes metrics (total queries, failures, average latency, and average throughput) using efficient, raw SQL database operations: db.execute(text(...)).scalar().
  • UI Grid Representation: Renders KPI cards and status badges at the top of the admin page, showing success, SQL failures, and guardrail denies.
  • Telemetry Logs Table: Renders a logs viewport displaying the last 10 execution query details (timestamps, latency, token count, and the exact generated SQL query).
